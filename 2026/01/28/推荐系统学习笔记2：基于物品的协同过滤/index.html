1. 构建物品流行度与共现矩阵

这个阶段的目标是统计用户行为，为后续计算相似度提供基础数据。

1.1 数学表达

  • 物品流行度 (Item Popularity)：指喜欢物品的用户数量。 其中，是指示函数，括号内条件为真则取1，否则取0。

    可以看到求和符号的下标是，所以在代码的实现中，是对每个用户进行遍历实现的。在循环中，再遍历每个物品，判断该物品是否被用户产生过行为（评分>0），有则计数。

  • 物品共现矩阵 (Co-occurrence Matrix)：指同时喜欢物品和物品的用户数量。 顾名思义，最终构建的是一个矩阵，是二维的。因此在代码中，使用permutations函数即可生成用户行为中的两两配对。

    例如用户A喜欢的物品有{a, b, d}，那么产生的配对就有：(a, b), (b, a), (a, d), (d, a), (b, d), (d, b)。根据产生的配对在共现矩阵对应的位置进行计数。和物品流行度一样，共现矩阵也是通过遍历每个用户实现的，每处理一个用户，共现矩阵就会更新一次。

2. 计算物品相似度矩阵

这个阶段利用共现数据来衡量物品之间的关联程度。

2.1 数学表达

这里使用的是基于条件概率的相似度公式。物品与物品的相似度定义为：在喜欢物品的用户中，有多少比例的人也喜欢物品。 和物品流行度不同，在代码中相似度矩阵的计算是对每个物品进行遍历，因此相似度矩阵的计算逻辑是一行一行循环下去的，即依次对每个物品计算它和其他物品的相似度。

因为相似度矩阵的分母是，即物品的流行度，直接获取模块一计算好的结果即可。

4.1 数学表达

初级公式的局限性在于：如果物品非常热门，很大，那么也会很大，导致偏高。为了平衡这一点，公式中引入了对物品流行度的惩罚： 其中：

  • ：惩罚因子，取值范围。
  • ：对起始物品的归一化。
  • ：对目标物品的惩罚项。物品越热门，分母越大，相似度被压得越低。

假设有三个目标物品等待被推荐：

  • X₁（小众配件）：100人喜欢
  • X₂（热门手机壳）：1000人喜欢
  • X₃（中等充电宝）：500人喜欢

用户历史：买了手机（物品A，10人喜欢）

A→X₁: 5人同时喜欢 (小众配件)

A→X₂: 50人同时喜欢 (热门手机壳)

A→X₃: 25人同时喜欢 (中等充电宝)

没有惩罚的相似度为：

w(A→X₁) = 5/10 = 0.5

w(A→X₂) = 50/10 = 5.0 ← 这个最高

w(A→X₃) = 25/10 = 2.5

有惩罚的相似度为：

w(A→X₁) = 5/√(10×100) ≈ 5/31.6 = 0.158

w(A→X₂) = 50/√(10×1000) ≈ 50/100 = 0.5

w(A→X₃) = 25/√(10×500) ≈ 25/70.7 = 0.354
