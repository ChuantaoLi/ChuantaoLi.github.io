基于关联规则的分类算法

目前使用较多且较为简洁的关联规则分类算法是基于关联规则的分类算法（Classification Based on Association, CBA），下面将从该算法的相关概念开始介绍。

这部分笔记参考论文：孙菡悦.基于多因素交互效应的农村贫困家庭精准识别研究[D]. 华东师范大学, 2022.

项集、频繁项集、关联规则

以一个小型的交易数据集为例：

交易ID (Transaction ID) 购买的商品 (Items)
1 牛奶, 面包, 黄油
2 面包, 黄油, 啤酒
3 牛奶, 面包, 可乐
4 牛奶, 面包, 黄油, 啤酒
5 面包, 可乐

项集 (Itemsets)

在这个数据集中，项集可以是不同商品的组合。我们可以从单个商品（单项集）开始，逐步扩展到多个商品的组合（多项集）。

  • 单项集（1-itemset）：{牛奶}、{面包}、{黄油}、{啤酒}、{可乐}
  • 二项集（2-itemset）：{牛奶, 面包}、{牛奶, 黄油}、{面包, 黄油}、{面包, 啤酒} 等等
  • 三项集（3-itemset）：{牛奶, 面包, 黄油}、{面包, 黄油, 啤酒} 等等

频繁项集 (Frequent Itemsets)

频繁项集是指在交易数据中出现频率较高的项集。我们可以设定一个最小支持度阈值，比如 60%（即项集至少出现在60%的交易中）。

  • 支持度 (Support) 的计算方法是项集出现的次数除以总交易次数。例如，项集 {牛奶, 面包} 出现了3次（交易ID 1, 3, 4），总交易次数为5次，所以支持度为 3/5 = 60%。

根据上述数据集，频繁项集可能包括：

  • {牛奶, 面包}，支持度 = 3/5 = 60%
  • {面包, 黄油}，支持度 = 3/5 = 60%
  • {面包}，支持度 = 5/5 = 100%

如果关联规则在发生频率不小于预先设定阈值的基础上同时符合最小置信度要求，则进一步定义该关联规则为强关联规则。

关联规则 (Association Rules)

从频繁项集中，我们可以生成关联规则，表示商品之间的关系。关联规则的形式为，表示如果购买了商品 A，则很可能会购买商品 B。关联规则有两个重要指标：

  • 支持度 (Support)：规则中项集的出现频率。例如，规则 {牛奶} -> {面包} 的支持度为 3/5 = 60%。
  • 置信度 (Confidence)：表示在包含前件的交易中也包含后件的交易比例。例如，规则 {牛奶} -> {面包} 的置信度为包含 {牛奶} 的交易中同时包含 {面包} 的比例。在包含 {牛奶} 的3笔交易（交易ID 1, 3, 4）中，有3笔同时包含 {面包}，所以置信度为 3/3 = 100%。该规则的置信度定义为在事件 A 发生的条件下，事件 B 发生的概率，用公式表示为：

  1. {牛奶} -> {面包}
    • 支持度：3/5 = 60%
    • 置信度：3/3 = 100%
  2. {面包} -> {黄油}
    • 支持度：3/5 = 60%

  3. {黄油} -> {面包}
    • 支持度：3/5 = 60%
    • 置信度：3/3 = 100%

Apriori算法

  1. 生成候选项集（Candidate Generation）：
    • 通过连接频繁项集生成候选项集。频繁项集的连接是通过对频繁项集中的项进行组合来完成的。例如，从频繁1项集生成候选2项集。
  2. 剪枝（Pruning）：
    • 利用Apriori性质：如果一个项集是频繁的，那么它的所有子集也是频繁的；如果一个项集是非频繁的，那么它的所有超集也是非频繁的。因此，可以在候选项集中删除那些包含非频繁子集的项集，从而减少计算量。
  3. 计算支持度（Support Counting）：
    • 通过扫描交易数据库，计算每个候选项集的支持度。保留支持度不小于最小支持度阈值的候选项集，作为频繁项集。
  4. 重复上述步骤：
    • 从频繁k项集生成频繁(k+1)项集，直到无法生成更多的频繁项集为止。

基于关联规则的分类算法过程

基于关联规则进行分类的基本思想是通过挖掘数据集中非分类特征（如属性、变量）与分类特征（目标类别）之间的关联规则，利用这些规则对新的样本进行分类。频繁模式反映了多个特征之间的关联关系，对分类具有显著影响。相比于决策树等只能考虑单一特征的方法，基于关联规则的方法能够同时考虑多个特征，从而提高分类的准确性和可扩展性。

  1. 挖掘频繁模式：
    • 从数据集中挖掘频繁项集，确定出现频率大于给定阈值的项集（频繁模式）。在这个步骤中，项集的后项（Consequent）是目标分类变量。例如，如果目标分类变量是“购买”与否，频繁模式可能是“年龄=30-40且收入=高” -> “购买=是”。
  2. 选出强关联规则：
    • 从挖掘出的频繁模式中选出支持度和置信度均满足条件的强关联规则。这些规则具有较高的支持度（Support）和置信度（Confidence），意味着它们在数据集中出现频率较高且具有较强的预测能力。
  3. 生成分类器：
    • 基于选出的强关联规则生成分类器。分类器的工作原理是将新的样本根据关联规则的前项（Antecedent）进行匹配，并将样本划分至对应的类别（后项）。

假设我们有如下数据集，目标是根据属性（年龄、收入、婚姻状况等）预测是否购买某产品：

ID 年龄 收入 婚姻状况 购买
1 30-40 高 已婚 是
2 20-30 低 未婚 否
3 30-40 中 已婚 是
4 40-50 高 已婚 是
5 20-30 高 未婚 否

步骤1：挖掘频繁模式

使用关联规则挖掘算法挖掘频繁项集，并生成候选规则。例如：

  • {年龄=30-40, 收入=高} -> {购买=是}
  • {婚姻状况=已婚} -> {购买=是}

步骤2：选出强关联规则

计算每条规则的支持度和置信度，筛选出支持度和置信度均满足设定阈值的规则，假设支持度阈值为0.4，置信度阈值为0.7。例如：

  • 规则 {年龄=30-40, 收入=高} -> {购买=是}
    • 支持度 = 2/5 = 0.4
    • 置信度 = 2/2 = 1.0
  • 规则 {婚姻状况=已婚} -> {购买=是}
    • 支持度 = 3/5 = 0.6
    • 置信度 = 3/3 = 1.0

以上两条规则均满足条件。

步骤3：生成分类器

基于强关联规则生成分类器。对于新样本，根据规则前项进行匹配，将其分类至对应的类别。例如：

  • 新样本 {年龄=35, 收入=高, 婚姻状况=已婚}
    • 规则 {年龄=30-40, 收入=高} -> {购买=是} 匹配，分类为“购买=是”
    • 规则 {婚姻状况=已婚} -> {购买=是} 匹配，分类为“购买=是”
