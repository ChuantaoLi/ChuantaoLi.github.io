兼容性处理

安装完成后，检查发现缺少了 apex。但安装时出现 --no-use-pep517 的报错。这是因为 Apex 仓库声明了 pyproject.toml 的构建后端， pip 不允许强行关闭 PEP 517，所以这条路被封死了。由于我的环境是 Python 3.10 + torch 2.11 + CUDA 12.8 较新，对项目的 apex_ddp.py 进行了修改，确保没有 Apex 就自动回退到 PyTorch Lightning 原生 DDP。修改后验证：

原有代码的学习率调度器和 torch 2.11 + lightning 1.6 组合有兼容问题，修改 train_cuboid_mnist.py 文件。具体而言，torch 2.11 下 Lightning 1.6 对 SequentialLR 的 API 校验不兼容，所以训练在优化器阶段报 MisconfigurationException 。解决方法是把 MovingMNIST 脚本改为单一 LambdaLR （warmup + cosine 合并），避免这个兼容冲突。
